## 一、总判断

需要整体化重构。

但这里说的“整体化”，不是把 5 个项目重写成 1 个项目。

真正要重构的是两层：
- 第一层：系统层
- 第二层：关键历史债

不建议现在做：
- 全量业务重写
- 全项目统一技术栈
- 全项目一次性迁移

#### 2. AdVideoSystem

目标：
- 先确认唯一主项目，再修跨平台硬伤

当前判断：
- `E:\\Project\\codex\\AdVideoSystem`
- `E:\\Project\\codex\\AdVideoSystem\\repo`

这两份是同一个 GitHub 项目的两份仓库。

证据：
- 两边 remote 一样，都是 `https://github.com/9ood/AdVideoSystem.git`
- commit 历史一样
- 外层 git 把 `repo/` 当成未跟踪目录
- 内层 repo 只是后来又拉了一份，并在里面补了 Windows 启动脚本、waitress、venv、ffmpeg 工具

结论：
- `repo/` 可以考虑删除
- 但现在不能直接删
- 正确动作是：先把 `repo/` 里真正有用的改动合并到外层主项目，再删除

必须先修：
- 明确外层目录为唯一主项目
- 把 `repo/app.py` 中的 Windows 启动增强合并出来
- 把 `repo/run_server.py`、`repo/scripts/*.ps1` 迁移到主项目
- 修复 [ai_analyzer.py](/E:/Project/codex/AdVideoSystem/ai_analyzer.py) 里的 `/Users/ssg/...` 写死路径

不建议现在直接做：
- 直接删除 `repo/`

建议删除条件：
1. 外层主项目已经补齐 Windows 启动脚本
2. 外层主项目已经补齐依赖和说明
3. 内层 `repo` 的差异已经迁移完
4. 已经做过一次启动验证

回滚：
- 删除前先把 `repo` 整个改名成 `_repo_backup_日期`
- 验证通过后再正式清理

## 五、AdVideoSystem/repo 的明确建议

结论一句话：

`repo/` 不是应该长期保留的结构，最终建议删除；但必须先迁移里面的有效改动，再删。

建议动作：

1. 把外层 `AdVideoSystem` 定为唯一主项目
2. 从 `repo/` 迁移这些内容到主项目：
- `run_server.py`
- `scripts/start_server.ps1`
- `scripts/install_startup_task.ps1`
- `requirements.txt` 里新增的依赖
- `app.py` 中对环境变量和缺失 API Key 的保护
3. 修复主项目中的绝对路径写死问题
4. 验证主项目可以独立在 Windows 跑起来
5. 把 `repo/` 改名为备份目录，观察一轮
6. 最后正式删除

## 六、通用化改造的总规则

以后所有项目都按这条总规则改：

- 业务代码通用
- 平台差异脚本化
- 配置外置
- 路径相对化
- 控制标准化

也就是说：
- 不再写“这个项目是 mac 专用”
- 而是写“这个项目业务通用，mac 和 Windows 各有启动脚本”

---

## 七、这份清单怎么用

你后面每次启动一个重构任务，都按这 5 个问题先问：

1. 这是系统层问题，还是业务层问题
2. 这是必须先修，还是可以后修
3. 这次只动一个项目，还是只动系统层
4. 验证命令是什么
5. 回滚动作是什么

只要一直这样做，后面新项目越来越多，也不会重新乱掉。
